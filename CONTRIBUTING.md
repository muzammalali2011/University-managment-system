# Contributing Guidelines

Thank you for contributing to the University Management System project!  
This project is structured with **frontend**, **backend**, and **database** components.  
To keep collaboration smooth, please follow these rules:

## Workflow
1. **Branching**
   - Create a branch for your work:
     ```
     git checkout -b feature/your-feature-name
     ```
   - Example: `feature/student-attendance-module`

2. **Commits**
   - Use clear commit messages:
     ```
     git commit -m "Add student attendance table"
     ```

3. **Pull Requests**
   - Push your branch and open a Pull Request (PR).
   - Request review before merging.

## Ground Rules
- No direct commits to `main`.
- Every PR must be reviewed by at least one collaborator.
- Respect coding style (PHP, MySQL, HTML/CSS/JS).
- Document database changes in `/docs/schema.md`.

## Security Updates
- Dependabot is enabled for dependency updates.
- Review and merge security PRs promptly.

## Collaboration
- Use GitHub Issues for bugs/features.
- Assign yourself to tasks you’re working on.
